Introduction

In this module, we will learn to:

  • tabulate levels
  • reorder levels
  • reverse levels
  • collapse levels
  • recode levels
  • recategorize levels
  • shift levels

Case Study

We will use a case study to explore the various features of the forcats package. You can download the data for the case study from here or directly import the data using the readr package. In this case study, we will:

  • compute the frequency of different referrers
  • collapse referrers with low sample size into a single group
  • club traffic from social media websites into a new category
  • group referrers with traffic below a threshold into a single category

Data

The data set we will use has just one column traffics i.e. the source of traffic for a imaginary website.

traffic

Data

Let us extract the traffics column from the above data using and save it in a new variable traffics. Instead of using traffic which is a tibble, we will use traffics which is a vector. We do this to avoid extracting the traffics column from the main data repeatedly.

traffics

Count

Count

Let us compute the traffic driven by different referrers using fct_count.

fct_count(traffics)

Levels

Use levels() to display the categories/levels.

levels(traffics)

Reorder

Reorder

The levels can be reordered based on their frequency using fct_infreq().

freq_traffic <- fct_infreq(traffics)
levels(freq_traffic)

Reorder

Reorder

The levels can be reordered based on the order of appearance using fct_inorder().

order_traffic <- fct_inorder(traffics)
levels(order_traffic)

Reverse Levels

Reverse Levels

The levels can be reversed using fct_rev().

reversed <- 
reversed <- fct_rev(traffics)
levels(reversed)

Collapse Categories

Collapse Categories

We want to group some of the referrers into 2 categories:

  • social
  • search

To group categories/levels, we will use fct_collapse().

traffics %>% 
  fct_collapse(
  social = c("facebook", "twitter", "instagram"),
  search = c("google", "bing", "yahoo")) %>% 
  fct_count()

Recode Levels

Recode Levels

Categories/levels can be grouped using fct_recode() as well:

fct_recode(traffics, search = "bing", search = "yahoo", search = "google",
  social = "facebook", social = "twitter", social = "instagram") %>%
  levels()

Lump Categories

Lump Categories

Let us group together referrer types that drive low traffic to the website. Use fct_lump() to lump together categories.

traffics %>% 
  fct_lump() %>% 
  table()

Lump Categories

We want to retain the top 3 referrers and combine the rest of them into a single category. Use fct_lump() and set the argument n to 3 indicating we want to retain top 3 categories and combine the rest.

traffics %>% 
  fct_lump(n = 3) %>% 
  table()

Lump Categories

Lump Categories

Let us combine referrers that drive less than 10% traffic to the website. Since we are looking at proportion of traffic driven to the website and not the actual numbers, we use the prop argument and set it to 0.1, indicating that we want to retain only those categories which have a proportion of more than 10% and combine the rest.

traffics %>% 
  fct_lump(prop = 0.10) %>% 
  table()

Lump Categories

What if we want to retain 3 referrers which drive the lowest traffic to the website and combine the rest? We will still use the n argument but instead of specifying 3, we now specify -3.

traffics %>% 
  fct_lump(n = -3) %>% 
  table()

Lump Categories

Let us see how to retain referrers that drive less than 10 % traffic to the website and combine the rest into a single group.

traffics %>% 
  fct_lump(prop = -0.1) %>% 
  table()

Replace Levels

Let us assume we want to retain a couple of important categories and group the rest into a single category. In the below example, we retain google and yahoo while grouping the rest as others using fct_other().

Replace Levels with Other

keep_gy <- 
keep_gy <- fct_other(traffics, keep = c("google", "yahoo"))
levels(keep_gy)

Replace Levels

What if you want to drop a couple of categories instead of grouping them? Use the drop argument in fct_other() and specify the categories to be dropped. In the below example, we drop the following referrer categories:

  • instagram
  • twitter

Replace Levels with Other

drop_it <- 
drop_it <- fct_other(traffics, drop = c("instagram", "twitter")) 
levels(drop_it)

Reorder Levels

The categories can be reordered using fct_relevel(). In the below example, we reorder the categories to ensure google appears first.

Reorder Levels

Similarly, in the below example, we reorder the levels to ensure twitter appears first irrespective of its frequency or order of appearance in the data.

move_twitter <- 
move_twitter <- fct_relevel(traffics, "twitter") 
levels(move_twitter)

Reorder Levels

If the category needs to appear at a particular position, use the after argument and specify the position after which it should appear. For example, if google should be the third category, we would specify after = 2 i.e. google should come after the 2nd position (i.e. third position).

Reorder Levels

appear_2 <- 
appear_2 <- fct_relevel(traffics, "google", after = 2)
levels(appear_2)

Reorder Levels

If the category should appear last, supply the value Inf (infinity) to the after argument as shown below.

Reorder Levels

appear_last <- 
appear_last <- fct_relevel(traffics, "facebook", after = Inf) 
levels(appear_last)

Shift Levels

Shift Levels

To shift the levels, we use fct_shift(). Use the n argument to indicate the direction of the shift. If n is positive, the levels are shifted to the left else to the right. In the below example, we shift the levels to the left by 2 positions.

shift_left <- 
shift_left <- fct_shift(responses, 2) 
levels(shift_left)

Shift Levels

Shift Levels

To shift the levels to the right, supply a negative value to the n argument in fct_shift(). In the below example, we shift the levels to the right by 2 positions.

shift_right <- 
shift_right <- fct_shift(responses, -2) 
levels(shift_right)
